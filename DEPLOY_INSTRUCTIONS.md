# Инструкция по деплою (публикации в интернет)

Поскольку проект использует базу данных SQLite (файл), его нельзя просто так залить на Vercel или Netlify (они удаляют файлы базы данных после каждого запроса).

Лучший бесплатный вариант для таких проектов — **Render.com**.

## Вариант 1: Самый простой (через GitHub)
Если у вас есть GitHub, это займет 2 минуты.

1. Создайте репозиторий на GitHub и загрузите туда файлы проекта.
2. Зарегистрируйтесь на [Render.com](https://render.com).
3. Нажмите **New +** -> **Web Service**.
4. Выберите ваш репозиторий.
5. В настройках укажите:
   - **Build Command:** `npm install && npm run build`
   - **Start Command:** `npm start`
6. Нажмите **Create Web Service**.

## Вариант 2: Glitch.com (Без GitHub)
Если не хотите возиться с GitHub:

1. Зайдите на [Glitch.com](https://glitch.com).
2. Нажмите **New Project** -> **Import from GitHub** (если есть) или просто создайте **glitch-hello-node**.
3. Вручную скопируйте содержимое файлов `server.js`, `package.json` и папки `src`.
   *(Это долго, поэтому лучше Вариант 1)*.

## Почему не Vercel?
Vercel идеально подходит для статических сайтов. Если вы зальете туда этот проект, он будет работать, но **все новые фильмы будут исчезать** через 10-15 минут, так как Vercel "перезагружает" сервер и стирает все созданные файлы (включая базу данных).

Для Vercel нужно переписывать базу данных на PostgreSQL (Supabase или Neon), что сложнее.
